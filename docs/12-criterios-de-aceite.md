# 12 — Critérios de aceite do MVP

## Home

- [ ] Carrega sem erro em desktop e mobile.
- [ ] Exibe hero editorial.
- [ ] Exibe pelo menos 4 seções de obras/capítulos.
- [ ] Possui busca visível.
- [ ] Possui links para login, explorar e para autores.
- [ ] Não depende de neon/gradientes fortes para ter identidade.

## Explorar

- [ ] Permite filtrar por gênero.
- [ ] Permite ordenar por recente/popular.
- [ ] Exibe estados vazios.
- [ ] Funciona em mobile.

## Página da obra

- [ ] Exibe sinopse, capa, autor, status e tags.
- [ ] Lista capítulos em ordem correta.
- [ ] Botão “Começar leitura” abre primeiro capítulo publicado.
- [ ] Botão “Adicionar à biblioteca” exige login ou salva quando logado.
- [ ] Metadados de SEO estão presentes.

## Leitor

- [ ] Texto legível em mobile.
- [ ] Navegação anterior/próximo funcionando.
- [ ] Ajuste de tema funcionando.
- [ ] Ajuste de fonte funcionando.
- [ ] Progresso pode ser salvo quando logado.

## Login

- [ ] Cadastro com validação de email e senha.
- [ ] Login com feedback claro.
- [ ] Erros não revelam informação sensível.
- [ ] Recuperação de senha prevista.

## Biblioteca

- [ ] Usuário consegue salvar/remover obra.
- [ ] Obras salvas aparecem em “Minha Biblioteca”.
- [ ] Último capítulo lido aparece, quando existir.

## Admin

- [ ] Admin consegue criar obra.
- [ ] Admin consegue criar capítulo.
- [ ] Admin consegue publicar/arquivar.
- [ ] Admin consegue definir destaque com data inicial/final.

## SEO

- [ ] Sitemap gerado.
- [ ] Title/description por rota pública.
- [ ] Canonical em páginas públicas.
- [ ] Imagens com alt.
- [ ] Estrutura semântica correta.

## Segurança/LGPD

- [ ] Senhas não são armazenadas em texto puro.
- [ ] Notificações exigem consentimento.
- [ ] Usuário pode alterar preferências.
- [ ] Termos e privacidade acessíveis no rodapé.
